# セキュリティ方針

このプロジェクトでは、セキュリティの問題を真剣に受け止め、責任を持って対処することをお約束します。

## 脆弱性の報告

セキュリティ上の懸念事項を発見した場合は、public GitHub Issuesではなく、以下のメールアドレスに直接ご連絡ください：

```
security@example.com
```

報告後、できる限り迅速に対応いたします。

## セキュリティ対策

- APIキーは `.env` ファイルに保存し、Gitにコミットしないようにします
- 依存関係の定期的な更新と脆弱性スキャン
- CORS設定によるセキュリティ強化
- ユーザー認証と認可の適切な実装
